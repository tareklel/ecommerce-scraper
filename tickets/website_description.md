# Ticket: Website Description — Owner Input Required

**Status: done — superseded by `context/product-brief.md`**

## What This Is

Before any frontend work begins, we need one paragraph that describes the website
as if explaining it to a first-time visitor. This description will be used to:

- Reshape `tickets/image_serving.md`, `tickets/product_api.md`, and
  `tickets/plp_debug_ui.md` so the technical decisions reflect the actual product
  rather than a generic price comparison tool
- Anchor the v0 design discussion (`tickets/frontend_v0_discussion.md`) — framework
  choice, layout, and information hierarchy all follow from what the site is trying
  to do for the user
- Serve as the reference point when making future prioritisation calls

## What to Cover

Write one paragraph (4–8 sentences) that answers:

1. **Who is the user?** (e.g. a luxury shopper in the Gulf, a deal-hunter, a
   fashion-forward browser, a gift buyer)
2. **What problem does the site solve for them?** (e.g. they want the best price
   on a specific item, they want to discover what's on sale across sites, they
   want to browse a curated selection without visiting five retailer tabs)
3. **What does the user do on the site?** (e.g. search for a brand or item,
   browse a category, set a price alert, compare the same item across two stores)
4. **What is the single most important thing the site communicates?** (e.g. price
   transparency, discovery, deals, curation, trust)
5. **What does it feel like?** (e.g. editorial and slow like a magazine, fast and
   utilitarian like Google Shopping, clean and minimal like a boutique)

## Example Format

> "[Site name] is a price comparison and discovery tool for luxury fashion in the
> Gulf. It brings together stock and pricing from [sites] so that shoppers can find
> the best current price for an item without visiting each retailer individually.
> The primary user is someone who already knows what brand or category they want
> and is looking for value — either the lowest price today or a discount on something
> they have been watching. The site should feel fast, clean, and trustworthy: closer
> to a well-organised search result than a retailer's own browsing experience."

This is a template — write your own version, not this one.

## What Happens Next

Once the description is written, bring it back here and we will:
1. Update the three blocked tickets to align technical decisions with the product intent
2. Open the v0 discussion with the description as the first input
